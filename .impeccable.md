# Impeccable Design Context

## Design Context

### Users
Solo developers who use OpenCode for AI-assisted coding. They check the dashboard to understand their usage patterns, track token consumption, monitor costs, and optimize their workflow. Context is personal desktop/server use—typically viewed during work sessions to understand real-time activity or after-the-fact to review spending.

### Brand Personality
Friendly, modern, accessible. The interface should feel welcoming and approachable rather than intimidating or overly technical. Like a well-designed consumer app, but with the precision developers expect from tooling.

### Aesthetic Direction
Minimal SaaS aesthetic inspired by Linear and Vercel—clean, purposeful, with generous whitespace and restrained use of color. Not playful or whimsical, but warm and human. Avoids the cold/clinical feel of enterprise dashboards.

### Design Principles

1. **Clarity over density** — Information hierarchy is paramount. Users should find key metrics (cost, tokens, sessions) immediately without visual clutter.

2. **Progressive disclosure** — Summary views are scannable at a glance; detail views are available on demand. The dashboard doesn't overwhelm.

3. **Warm neutrals, not stark contrasts** — Use soft grays and warm tones rather than pure black/white. The interface should feel like a comfortable workspace.

4. **Consistent micro-interactions** — Subtle transitions and hover states make the interface feel alive and responsive. Avoid jarring state changes.

5. **Data visualization with purpose** — Charts serve understanding, not decoration. Every visualization should answer a question the user has.
